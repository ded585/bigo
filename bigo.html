Big-O

  • How code slows as data grows
  • Mathy, but doesn't have to be
  • Rough characterization

Terminology

  • N: how much data
  • O(blah blah N blah)
  • "Order of"
  • Not a function!

Finding words

Other terms

  • O(1): constant time
  • O(N): linear
  • O(N2): quadratic

Determining Big-O

  • Identify your code
  • Identify N
  • Count the steps in a typical run
  • Keep the most significant part

An Example

    moms = [
        ("Ned", "Eleanor"),
        ("Max", "Susan"),
        ("Susan", "Shelly"),
        ...
    ]
    def find_mom(moms, child):
        """Find a the mom of `child`."""
        for child_name, mom_name in moms:   # 3 * N/2
            if child == child_name:         # 1 * N/2
                return mom_name             # 1
        return None

3N/2 + N/2 + 1 →  2N + 1 →  O(N)

Another example

    def how_many_grandmothers(moms):
        """How many moms are grand-moms?"""
        grandmothers = 0
        for child, mom in moms:             # 3 * N
            grandma = find_mom(moms, mom)   # N * N
            if grandma:                     # N
                grandmothers += 1           # kN
        return grandmothers                 # 1

N2 + (4+k)N + 1 →  O(N2)
